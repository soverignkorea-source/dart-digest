# DART 공시 주간 다이제스트

다트해킹(@darthacking) 텔레그램 채널의 한 주(월~금) DART 공시를 **요일별 + 공시목적별**로
자동 정리해 다크 테마 HTML로 만들고, **GitHub Pages 고정 URL**로 매일 갱신합니다.

## 화면 구성
- **주간 요약 칸** : 그 주 중요도 ★★ 이상 공시를 한 줄씩 모아 상단 표시.
- **탭: 주간 전체 / 월 / 화 / 수 / 목 / 금** : 요일별로 그날 공시를 목적별로 정리. "주간 전체" 탭은 한 주 전체를 목적별로 모아 보여줌(카드에 요일 배지).
- **목적별 한눈에** : 카테고리 칩을 누르면 해당 카테고리 섹션으로 바로 스크롤.
- **중요도 별점 ★~★★★** : 주가 영향도를 자동 추정해 카드마다 표시.
  - ★★★ 유상/무상증자·전환사채, 자사주 소각(시총3%+), 최대주주변경, 상장폐지·거래정지·감사의견·관리종목, 매출대비 30%+ 대형수주, 매출 10%+ 영업정지
  - ★★ 매출 10~30% 수주, 자사주 취득/소각, 경영권 영향 지분변동, 투자판단·임상·기술이전, 인수/매각 보도 해명, 주식병합
  - ★ IR·기업설명회, 조회공시 답변, 단순투자 지분보고, 소형 수주 등

## 동작 방식
- `scripts/generate_digest.py` : 채널을 그 주 월요일 0시부터 긁어 → 9개 목적 카테고리 분류 + 중요도 산정 → `index.html`(이번 주) 생성, `archive/연도-주차.html` 보관.
  - 매 실행마다 **그 주 전체를 다시 만들기** 때문에, 주중 매일 돌리면 요일 탭이 하루씩 채워짐. 월요일이 되면 자동으로 새 주가 시작되고 지난 주는 아카이브에 남음.
- `.github/workflows/daily.yml` : 매 평일 **19:30 KST**(장 마감 후) GitHub 서버가 자동 실행 → 결과를 저장소에 커밋. PC 불필요.


## 지난 주 열람 / 백필
상단 **주차 선택기**로 지난 주 다이제스트를 바로 열 수 있습니다(최초 설치 시 과거 데이터를 함께 채우려면 아래 실행).
```
python3 scripts/generate_digest.py --backfill 4   # 이번 주 + 지난 4주
```
매일 자동실행은 이번 주만 갱신하며, 지난 주들은 아카이브에 그대로 남습니다.

## 설치 (한 번만)
1. 깃허브에서 **새 public 저장소** 생성 (예: `dart-digest`).
2. 이 폴더 파일 전체 업로드.
3. **Settings → Pages** → Source: *Deploy from a branch*, `main` / `/(root)` → 저장.
4. **Settings → Actions → General → Workflow permissions** → *Read and write* 체크.
5. **Actions 탭 → Daily DART Digest → Run workflow** 로 첫 실행.
6. `https://<아이디>.github.io/dart-digest/` 즐겨찾기.

## 수집 시각/요일 변경
`.github/workflows/daily.yml` 의 cron(UTC, KST=UTC+9). 예) 주말 포함 20:00 KST → `0 11 * * *`

## 과거 주 백필
```
python3 scripts/generate_digest.py 2026-06-18   # 해당 날짜가 속한 주(월~금)
```

> 별점은 자동 산정한 참고 추정치입니다. 본 다이제스트는 정보 정리 목적이며 투자 권유가 아닙니다.
